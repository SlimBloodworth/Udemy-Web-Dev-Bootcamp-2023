Primitive Types in JS

Primitives are the basic building blocks.

  • Numbers
  • Strings
  • Boolean - true or false values
  • Null
  • Undefined
  • Symbol and BigInt - not used as often.

The Console

A place to type code. That code is then run and then is repeated back/result is shown to you. This is known as:

R.E.P.L. - Read, Evaluate, Print, Loop

Numbers

JS has 1 number type

Math Operators

  • Addition: +
  • Subtraction: -
  • Multiplication: *
  • Division: /
  • Modulo: %

PEMDAS

Order of Operations

  • P - Parenthesis ()
  • E - Exponents 32 used as 3 ** 2 = 9. Takes the first number and raises it to a power.
  • M - Multiplication *
  • D - Division /
  • A - Addition +
  • S - Subtraction -

This is the order in which arithmetic will be done.

Example:

(3 + 1) * 9

The equation within the parenthesis will be done first, then that sum will be multiplied by 9.

The output is 36, which is 9*4

Modulo %

This operator gives the remainder of a division.

Example: 9 % 2 = 1. 2 goes into 9 4 times, the remaining amount is 1: 2 * 4 = 8 and 8 + 1 = 9, or 9 - 8 = 1.

Mostly used to determine is a number is even or odd. If the remainder = 0, this means the number is even.

NaN - Not a Number

A numeric value that represents something that is not a number.

Examples: 0/0 = Nan, undefined. 1 + NaN = NaN

NaN and any operator will equal NaN.

NaN is considered in the number family in JS. typeof Nan = number

JS Numbers Quiz Section 14: Lesson #147

What does this evaluate to?

4 + 3 * 4 / 2

PEMDAS

no P, no E, M is 3 *4 = 12, D is /2... so 12/2=6, A is 4 +, so 4 + 6 = 10, no S.

This evaluates to 10.

What does this evaluate to?

(13 % 5) ** 2

PEMDAS

P is 13 % 5 = 3, E is ** 2. 3 ** 2 = 9, no M, no, D, no A, no S.

This evaluates to 9.

What does this evaluate to?

200 + 0/0

PEMDAS

no P, no E, no M, D is 0/0 = NaN, A is 200 +, no S.

200 + NaN = NaN

Variables - Section 14: Lesson 148

Variables are like labels for values.

We can store a value and give it a name. The we can refer back to it later, use it to do things, or change it.

Basic Syntax

let variableName = value;

Make a variable named "year", and give it the value of 1985.

let year = 1985;

This is called declaring a variable.

Coding Exercise 15: Variables

The Challenge:

Define 2 variables: one called myLuckyNumber and set the value to 37.

The Other called octopusLimbs and set the value to 8.

The Solution:

let myLuckyNumber = 37;

let octopusLimbs = 8;

Section 14: Lesson 149 - Updating Variables

let score = 5; The value oof score is now 5.

score = score = 5; The value of score is now 10.

A shorter syntax is: score += 5; Ths takes the current value stored in score, sets it equal to that value and adds 5.

If the value of score was 10, score += 5; updates the value of score to 15.

This works the same for the other math operators as well, if the value of score is 10, score -= 5; updates the value of score to 5.

Incrementing and decrementing are very common, so there is a shorter syntax for that:

let score = 9;

score++ will increment by 1, now score is = 10.

score-- will decrement by 1, now score is = 9.

Section 14: Lesson 150 - Increment Operators

i++ vs. ++i

The ++ operator adds 1 to a number and returns a value, BUT - it can return a value before or after the increment, depending on the actual placement of the ++ operator.

i++ - the post increment operator.

example: let variableName = 0;

variable++; - returns 0, THEN variableName returns 1 when entered alone.

variableName; - returns 1.

When used, it returns the current value of that variable and then increases by 1.

The i++ expression was executed, but returned the value PRIOR to the increment, then updated the variable after.

++i - the pre-increment operator.

This increments the variables value by 1 first, then returns the incremented value after that.

Section 14: Lesson 151 - Const and var

const: Works just like let, except you CANNOT change the value.

You cannot change it, re-assign it, or increment it.

example: const cats = 4;

cats = 20; //returns an error.

const age = 18;

age = age + 1; //returns an error.

A const variable remains constant. You cannot update the value or change the value.

Useful for: storing values that we either know won't change, or don't want to change such as arrays and objects.

var: at one point in time var was the only way to declare a variable, but is not used as often now.

Var works very similarly to let, but there are some distinctions.
